What is data

Entities

An entity is a data-based representation of a real-world object (a person), and an attribute is a specific piece of information belong to an entity (a name)

How is data organised

a data store: Acts as a repository to hold an organization's data After filling it with data a data store becomes an archive that can then be searched.

Where data comes from

Records, fields and metadata : A record is the set of data stored about a particular instance of an entity. A field is a single piece of data in a record that corresponds to an attribute. Metadata is any data that describes other data. How do we use it? Use records, fields and metadata to record and describe the contents of a data store

Aquiring data

Manual data entry or a bulk import

CRUD operations

Create Read Update Delete operations to be carried out against a data store

Data Access Patterns

Ways of interacting with a data store, a pattern of interation on the data stores primary usage, based on applications needs.

Data store types

Data store types : file based / databases

Files based

data stored in simple files Use it when working with simple data such as data exported from Excel sheets or other comma separated value (CSV) formatted data

JSON & XML

Key Value Pairs: Use XML or JSON structure in particular when data originates in nested formats

Databases :

A databsse management system :A DBMS manages all access to the data providing security reliability ease of administration and performance, Use a DBMS when working with data that requires rapid retrieval and modification

Context of a database in your software

Using data as part of an application . Data provides the meaning for your application's existence How? Store data at rest in a DBMS. Manipulate data in motion as part of executing the application's business rules. Represent data as presented in the user interface (UI)

Relational Databases SQL structured query language

Represents data in table form. Relationships can be created and maintained among tables. Use a relational database for highly structured, non-nested data formats

No SQL: Document oriented Databases MongoDB etc

A non-relational DBMS: Provides a means of working with high volumes of non-relational data usually in JSON or XML format: Use a document-oriented DBMS as a way of grouping collections of similar formatted document data.

NoSQL And Key-Value Databases

Key-value DBMS : Stores data in key-value pairs when the actual data (the value) can be identified by an associated key. : Use a key-value based DBMS when working with data that doesn't have a very well defined structure

Specialised Databases & Summary Of Overview

Specialist database systems : Provide specialization or niche storage formats : Consider these options based on very specific requierements and where an RDBMS is not effective

  • graph database ie facebook suggested friends
  • timeseries database ie stock prices
  • geospatial database - maps

Conceptual Design Phase -IMPORTANT for any dev project.

A series of activities in a database design process. Determines all the entities that are important to business implementing business rules. Represent your application data at a conceptual level by creating an Entity-Relationship (ER) data model. many to many one to many ! remember hydrasound how i used that with studios events etc. This is an example of appoaching design phase.

Logical Design Phase

A set of activities that take place on the outputs of the conceptual design phase. The process of defining the higher level entities to be stored as tables. It is a refining of the ER model created during the conceptual phase. Use the logical design phase to determine the data types, primary and foreign keys to be used among the tables

examples:

  • int
  • varChar
  • text
  • Bool

Null Values

A representation of data when it's type is unknown

Allows the creation of a record where the expected data type in a field can be empty. Use the null value type when there is the possibility of a record field having an empty value.

Physical Data Model

The last and most concrete phase of the ER modelling process Considers how the tables in our database will be stored and accessed Use this to determine what server hardware and indexes might be required.

Normalisation

A process of optimizing the organization of data Normalization is concerned with breaking up data into tables to stop it from being redundant Apply normalization to reduce unnecessary disk space usage, speed up data access and minimize the risk of corrupting your data.

SQL

It allows the communication and the provision of instructions to a relational database Use SQL to create, access and modify a database, it's tables, records and fields.
